ILM Policy Builder

Redesigning Identity Lifecycle Manager Policy Builder through comprehensive user research and usability validation. Led research with enterprise customers across healthcare, government, and utilities sectors to understand policy management workflows, identify UX pain points, and validate design solutions that reduce complexity in bundle and policy deployment.

Design Process

Research & Strategy

Challenge

The existing Policy Builder relied on cumbersome XML editing, requiring deep technical expertise and consulting support. Administrators struggled with fragmented workflows spread across multiple modals, which disrupted focus and increased cognitive load. The web-based XML editor lacked direct editing capabilities and didn't match how modern administrators expected to work. The core problem: how do we transform a complex, technical XML tool into an intuitive interface that reduces reliance on experts while maintaining power for advanced users?

Approach

  • Conducted contextual inquiry sessions with system administrators to understand policy workflows
  • Documented core pain points: XML editing complexity, fragmented modal-based workflows, and outdated interface limitations
  • Defined 8 research objectives covering ease of use, efficiency, error handling, navigation clarity, and user satisfaction
  • Designed and validated Canvas View, XML Tree View, and multi-view interaction patterns through iterative testing
  • Conducted Innovation Lab usability testing with 9 customers and partners for real-world validation

Solution

Redesigned the IDM Policy Builder from a cumbersome XML-based interface into an intuitive, visual policy editor. Introduced a Canvas View for guided rule creation with a node-to-node structure, enhanced XML Tree View for navigation, and multiple viewing modes (Canvas, XML Tree, XML Source) allowing users to pivot between visual and technical representations seamlessly. The design reduces reliance on deep XML expertise while maintaining full control for advanced administrators.

Outcome

  • Documented 50+ actionable UX improvements with customer validation
  • Roadmap prioritization informed by real user needs across enterprise segment
  • Stronger customer relationships through collaborative research and validation process
  • UX research methodology established for future enterprise product development
Design Features

Key Interface Innovations

Canvas View

The centerpiece of the redesign. Users create and organize rules, conditions, and actions in an intuitive, card-based visual layout. The canvas displays a node-to-node structure showing relationships between elements, enabling users to visualize complex, nested policy structures all in one unified workspace.

XML Tree View (Enhanced)

Appears in a sidecar panel for efficient navigation. Users click on nodes to view details on the canvas, with interactive highlighting showing exactly where each condition or action fits within the overall policy structure. Side-by-side navigation enables seamless editing without losing context.

Multi-View Toggle

Top and lower panel toolbars provide instant access to three perspectives: Canvas View (visual), XML Tree View (hierarchical), and XML Source View (technical). Users seamlessly pivot between views based on their task and expertise level.

Guided Rule Creation

Contextual instructions and empty states guide users through creating rules with attached policies. Users can view multiple rules on the same canvas and pivot to deep-dive nested rules while maintaining a cohesive workflow. Even complex rules become manageable.

Interactive Highlighting

Selecting a condition or action in Canvas Mode automatically highlights its corresponding location in the policy structure. Users immediately see relationships and can make informed adjustments with confidence.

Reduced XML Dependency

Administrators can now create, edit, and manage complex policies without deep XML knowledge. The visual interface eliminates reliance on consulting support while maintaining technical depth for power users who need it.
