Oracle Unity Data Platform

Complete User Journey: From Data Ingestion to Exploration

Ingesting Data
👤 Oracle System Administrator Gavin — manages job configurations and data source connections
Core Tasks:
Design an ingest job object
Create source instances (SFTP, S3, Object Store)
Map external data to OUDD tables
Run publish job to commit data
Goal: Source data gets into the OUDD data warehouse via design-time configuration that becomes run-time ingest jobs.
Mastering Data
👤 Customer Instance Administrator Configures and manages data quality and enrichment

👤 Power Users & API Developers Consume and build on mastered data
Operations:
Unify, augment, and promote data
Run Analytics Engine jobs
Analytics Engines:
Relationship Builder — Identifies related records
Data Augmentation — Enriches data attributes
Master Promotion — Publishes golden records
Goal: Raw ingested data becomes clean, unified, and enriched through mastering processes.
Using & Exploring Data
👤 360 Client Users Enterprise data consumers

👤 Basic Data Explorers Ad-hoc query and reporting users
Actions:
Create sink instances for data export
Generate UI widgets and adhoc queries
Export data for downstream systems
Capabilities:
OUDD Queries (JSON-based)
Reporting on mastered data
OUDD 360 View (ID Graph)
CSV file exports
Goal: Mastered data is now discoverable, queryable, and available for reporting and business analytics.
Extract & Load
Raw data ingested from sources
Transform & Enrich
Data unified and augmented
Activate & Serve
Data available for consumption
🎯 Strategic Value

The Admin UI (Job Configs, Source Instance Management, Column Mapping) is the critical first step that enables the entire platform. Without Gavin's ability to efficiently ingest and configure data, downstream mastering and exploration use cases cannot succeed.
